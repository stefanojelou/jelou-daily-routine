🆕 Builder is ~2x stickier than agent chat — conversation activates wide but shallow, building activates narrow but deep
Post-activation week-over-week self-retention diverges sharply by surface (prod, born May 25–Jul 1, weekly curves). Once a user first BUILDS (node_used), 43% return the next week and retention plateaus ~32% through week 6. Once a user first MESSAGES an agent (agent_message_sent), only 24% return week 1, decaying to ~16% by week 6 — a sustained ~1.8x (wk1) to ~2x (wk6) stickiness gap. This complicates the Jul 17 read that agent conversation is the 'right' activation yardstick: conversation is the WIDER net (Jul 17: 65% of signups ever message vs 13% ever build) but the SHALLOWER hook per user, while building reaches few but holds ~2x better. Because reach differs so much, agent still yields ~3x more week-1-retained users in absolute terms (0.65×0.24≈15.6% of signups vs 0.13×0.43≈5.6%) — so neither 'conversation is the yardstick' nor 'building is stickier' is the whole story: conversation wins on volume of retained users, building on depth per user.

Sources: mixpanel
Confidence: medium-high (weekly retention curves, prod; builder avg cohort n~61/day, agent n~97/day; gap stable across daily cohorts wk1–wk6)
Caveat: Event-age confound: node_used has long pre-window history so born-in-window builder cohorts may include seasoned users re-engaging (inflating builder stickiness), whereas agent_message_sent launched ~May 18 so its births are near-true-first — the gap DIRECTION is robust (large, stable across daily cohorts) but the MAGNITUDE may be overstated; a true first-ever-builder cohort needs DB confirmation.
